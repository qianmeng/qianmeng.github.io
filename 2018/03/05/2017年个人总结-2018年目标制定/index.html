2017年工作回顾

一年时间稍众即逝，不知不觉又到过年了，回到家感觉上年也就刚过去。但回想上年都做了什么事的时候，突然脑子一片空白，想不起一年到底干了些什么，这种感觉差爆了，人的人生也就几十年，就这样一年一年过去了，并没有感觉到自己的成长，这是很严重的问题，所以现在尽量回想上年，只有回想起才可以总结自己的问题，下年针对性的补缺（虽然仅仅补缺已经是落后了），才能过得更充实，不至于碌碌无为。
幸运的是有几点踪迹参考进行回想上年的经历

1、个人博客

最新几篇都是面试题总结篇。

2、外派工作

发现并指出另一个公司的项目存在的问题，建议项目使用微服务体系架构。

3、跳槽工作

跳槽到另一家公司，接触新项目。

4、个人房产证

通过查看自己的朋友圈，本人于2017年9月份拿到了自己在南京的第一套房子。

  • 个人博客不知几何时就逐渐停止跟新原创文章了（本身也没有写多少），后面觉得一些文章比较不错，而且可以珍藏使用，就以转载的方式放到自己博客上了。由于工作前段时间招了不少新人，本人也参与了面试，并承担技术问答方面的工作，于是整理了各方面技术的面试常用问题（虽然面试并用不了那么多度，而且面试正常已项目实践为主，整理也同时检验自己的基础知识）。
  • 外派工作主要以技术顾问的身份找出项目中存在的问题，并建议项目使用微服务体系架构。微服务体系架构也是2017年初自己涉及到的体系架构，由于公司系统的不断复杂、微服务概念的提出、Spring
    boot、Docker、CI/CD等新技术框架的出现，公司准备实施技术转型，转型的初始版本由我来负责，也花了不少时间、精力，深入了解的各方体系架构，理解微服务体系架构的优缺点以及适用的应用场景。
  • 由于各种机缘巧合，本人于2018年1月1日进行了自参加工作以来的第一次跳槽行为，虽然不确认之后的旅途是否顺畅，但也是一次不一样的体验与挑战。进入新公司也是因为看到了公司发展的前景，与自己进入的时机恰逢其时。毕竟在之前的公司工作了长达六年，工作习惯与工作方式与之前公司确实有所不同，当然这也是必然，现阶段在慢慢适应中，新公司对本人个人技术还是要求蛮高的，庆幸的时相关技术都有接触过（微服务、大数据），虽然不是业界精通，也算是有方向，知道如何学习拓展自己的能力。

确定2018年各类小目标。

工作学习方面：

1、博客

博客是总结自己技术不错的方式，同时可以分享给其他同行技术人员，达到互相学习，提高技术水平不错的途径，如果继续高技术，博客是不能丢的，而且要经常写，养成习惯，后续为了督促自己，而且在年末的时候自己意识到技术上的欠缺，以后其买了五百多块钱的书籍，这是之前没有过的行为，自己也意识到了读书的好处，相比整天搜搜百度啥的，读书能够更为系统的理解一门技术，而且以后可以作为工具进行问题查找，这样长期下来，才能在技术方面做到专业。先给自己定个小目标，争取一周一篇博客，内容可以纯技术、读书总结、工作总结，最终目的为锻炼写作水平，总结知识。

2、技术

技术不能丢，不仅仅是为了对公司技术的把关，对最新技术的不断学习，避免被飞速发展的社会遗弃，也是为了自己今后的工作方向打下坚定的基石。搞技术的人都知道，一周不写代码，再次开始写的时候会感觉很生疏，一周不关注最新技术动态，可能要落后不少，为了技术不能丢，之后的工作中，经常关注一些场景，如果通过代码能够提高工作效率的情况下，就使用代码来实现，并将代码封装上传到github上，供后续改善使用，如果有需要，可以买个低配的阿里云进行代码效果实验，同时可以通过外网随时访问（之前也是因为在乎钱，同时也是因为没有可放置的内容，所以一直没买）。大数据技术风靡的时代下，不能落伍，大数据不是一般代码简单环境就可以试验的，所以之后的第一要务是采用虚拟机的方式，将大数据环境搭建起来，后续做一些练习使用，大数据、机器学习、深度学习、神经网络、强化学习，这些技术很多专家也说过，不能只看些理论，一定要上手做，不然真的没啥效果，例如一些经典的算法，朴素贝叶斯法、支持向量机、最近邻居法、决策树等等，如果不用程序自己实现一遍，真的不会深入理解。

3、工作技能

公司的环境决定自己的工作方向，有些东西公司不会下给你明确的指示告诉你需要学习什么知识，给你多长时间去研究一些技术等等，更多的时候是临时有些工作安排需要你去解决，有时并不是你所擅长的技术方面的工作，所以你要全方位的掌握一些工作中需要用到的技能，包括并不限于工作汇报文件、技术方案、业务需要、人员协调、数据整理等等。有些其实并非技术专业技能，但有时发现长时间钻进技术研究，反而疏忽了一些常用的技能，例如文档编辑、整理（Word、Excel的常用技巧），网络布线（公司网络坏掉问题所在、跨网是否可以联通，如果不能采用什么手段解决），人员沟通需要注意事项、领导没有明确任务的前提下，如何进行工作等等。所以以后这方面要比较重视，有些技能在特定环境下不需要，但属于通用技能，早晚会遇到，到时候被别人指责常识性知识都不会就尴尬啦。

4、业务技能

业务和技术要共同兼顾，虽然现在公司给我的定位是技术，但我感觉至少现在的工作内容已经不限于只搞技术了，而且以我之前的工作经验，虽然没有带过一些比较大型的项目，但也算是个项目经理了，所有现在也是一个机会，同时兼顾业务和技术，理论上不难，但我的业务分析水平是有欠缺的，这点还是有自知之明的，但也已经意识到了，业务是要有一些行业经验的，比起技术，也许要比技术更难掌握，但大多时候也是业务内容也是相同的。业务在一个产品中的地位是非常重要的，一定要掌握项目的整体业务才算是完整的，所以以后要作为重点方向却学习，提升自己。

5、表达能力

个人能力欠缺的方面还是不少的，到新公司暴露比较明显的问题主要是语言表达十分欠缺，之前工作环境基本上是纯搞技术的，当时人与人交流的内容基本上是技术，而且之前的人员基本上没有等级之分，所有交流起来非常随意，有时候经常说些不正经的话，互相挑逗等等，但在新环境下，发现之前的交流方式完全不适合，有时候会因社会阅历不足插不上话，有时候想说一些自己知道的事，却组织不好话语，生怕在领导面前说错话，紧张混乱，思路条理不清，这是现在比较大的问题，所有以后这方面要着重培养，之前有人提供发方法，每天对着镜子找个主题，有条不紊的表达出来，长时间这样练习下去，肯定有效果，在自己没有其他更好的方法的前提，准备尝试的去实际操练下。

6、个人定位

个人定位问题，不能仅限于任务的安排，那样的工作会让自己堕落，缺少一定的激情，当然也要视自己能力而定，之前有位启蒙领导说过，当你是一位程序员的时候，你要站在项目经理的角度想问题，当你是一个项目经理的时候，你要站在公司领导的角度想问题，这样的你在公司才有发展机会，公司需要这样的人，所有以后要多想想公司的发展方向，站在领导的角度想问题，这样你的工作才会顺利，工作才会有激情，有挑战。

私人问题方面：

通过查看自己的朋友圈，本人于2017年9月份拿到了自己在南京的第一套房子。虽然坐落于六合，还是比较偏的，但已不是之前在南京漂泊而是有种踏实的感觉了。房子从过年回来就一直开始找房产中介，自己去实地看房，花了很多时间精力，最终在一次去参加同事 婚礼的途中发现六合的发展还是蛮适合居住的，唯一的缺点就是离市区太远（地铁1个小时的车程），但由于资金情况，为了在南京能安顿下来，也怕南京的房价继续往上涨（2016年较2015年整整翻了一番），自己后面无法承担房价，这里是不错的选择，这里的小区全是新房，离地铁骑自行车10分钟路程，小区旁边有个很大的购物广场，再旁边是一个湖，地铁站的名字就是以这个湖命名-龙池。房子的大小也很合适，但里面的家具，可能要在2018年进行换掉，因为2018年，自己要对自己的终身大事上点心了，自己以后有女朋友了，而且是奔着结婚去的那种，女朋友非常优秀，2018年，我们准备慢慢往婚礼上进行准备了，这方面我们会按照既定的节奏走，在家人的大力支持下，希望能够一帆风顺。

2018年目标总结

  • 1、博客一周一篇，内容视情况而定。
  • 2、代码要经常写，内容可为日常工作工作中遇到的问题，也可通过程序完成自己的一些小想法；学习一些自己欠缺的而在工作中会用到的技术
  • 3、业务在一个产品中的地位至关重要，一定要了解一个项目的整体业务流程、方向，同时要学会分析项目业务线，设计相关流程。
  • 4、多方位完善自己的工作技能，包括为人处世，工作常用技能
  • 5、经常找一些主题，对着镜子或者对着一个人表达自己想要说的话，锻炼自己的语言组织能力。
  • 6、将自己的定位提高一个档次，不能仅限于任务的安排，要多想想公司的发展方向。
  • 7、2018年尽量完成自己的终身大事，前提就是把房子里的家具准备好，姻缘方面跟着节奏走，有家人的大力支持，希望能够一帆风顺。

目标是定下来了，虽然看上去不是很大，但之前也不是没有定过类似的目标，由于过程中各种原因，总是没有完成，或者过程中都忘了自己有定过目标，之前看到过一篇文章也评论过这种情况，其实这就是成功人士与普通人的区别所在，也许这就是自己一个普普通通的原因所在吧，好了，找到原因了，对症下药吧，既然有颗想成功的心，那就尽自己的全力去实现这些小目标吧，理想总是要有的，万一实现了呢！！！
